California Census¶

Se pretende ilustrar la regresin utilizando SVMs aplicando tanto la $\epsilon$SVR como la $\nu$SVR sobre los datos del censo de California por grupos de bloques de vivienda con el objetivo de predecir los precios medios de la vivienda. Tambin se deber de utilizar una red neuronal artificial para resolver el problema.

La variable dependiente es $ln(median\_house\_value)$, y se tiene que ajustar el siguiente modelo:

$$ln(median\_house\_value) = a_1 + a_2 \cdot median\_income + a_3 \cdot median\_income^2 + a_4 \cdot median\_income^3 + a_5 \cdot ln(median\_age) + a_6 \cdot ln(total\_room / population) + a_7 \cdot ln(total\_bedrooms / population) + a_8 \cdot ln(population / households) + a_9 \cdot ln(households)$$

6. Árboles de decisión ¶

Los rboles de decisin son algoritmos de aprendizaje supervisado donde es posible visualizar el rbol de decisin entrenado, por lo que la interpretacin de los resultados es ms fcil. La forma en la que se genera un rbol de decisin es cogiendo el mejor atributo del conjunto de datos y situndolo en la raz del rbol, para hacer un split y obtener un subconjunto de datos ms pequeo. Posteriormente, se realizan estos mismos pasos en cada subconjunto de datos hasta encontrar los nodos hoja en cada una de las ramas del rbol.

Los rboles de decisin, aunque son fcilmente interpretables, sufren por tener una alta varianza y un pequeo cambio en los datos, puede cambiar el resultado final del rbol. Por este motivo, tambin se han realizado modelos con Bagging, Random Forest y Boosting, que veremos ms adelante.

7. Bagging ¶

Bagging, o Bootstrap Aggregation, es un procedimiento de propsito general para reducir la varianza de un mtodo de aprendizaje estadstico. Hay que tener en cuenta que dado un conjunto de n observaciones $Z_1, ..., Z_n$, cada una con varianza $\sigma^2$, la varianza de la media $\overline{Z}$ de las observaciones est dada por $\sigma^2 / n$, es decir, con el promedio del conjunto de observaciones se reduce la varianza.

Por lo tanto, Bagging utiliza Bootstrap para coger varias muestras de un nico conjunto de entrenamiento y as generar B conjuntos de entrenamientos de Boostrap. Gracias a esto, y en combinacin con los rboles de decisin, genera B rboles de decisin y hace el promedio del resultado de las predicciones reduciendo la varianza.

8. Random Forest ¶

Random Forest provee una mejora sobre los rboles de decisin de Bagging para reducir ms la varianza. Esto se hace de modo que cada vez que se hace una divisin en un rbol, se considera una seleccin aleatoria de m predictores elegidos como candidatos de divisin de todo el conjunto de p predictores.

Normalmente se usan $m = \sqrt{p} \quad$ o $\quad m = p$, y se pueden indicar en el siguiente algoritmo mediante el parmetro max_features = 'sqrt' o max_features = 'auto', respectivamente. Adems, con el parmetro n_estimators indicamos el nmero de rboles que se generan y con max_depth la profundidad de cada rbol. Estimamos la mejor combinacin de estos parmetros mediante GridSearchCV.

10. SVM ¶

In [53]:
from sklearn.svm import SVR, NuSVR

A continuacin, se prueban tres modelos de $\epsilon$-SVM con tres kernels diferentes: lineal, Radial Basis Function (RBF) y polinomial de grado 3, y un modelo de $\nu$-SVM con kernel polinomial. Los parmetros para los cuatro modelos son los mismos, exceptuando el parmetro gamma para el RBF y el parmetro degree para el polinomial.

Los parmetros a analizar mediante GridSearchCV son:

  • C: parmetro de penalizacin del trmino de error. Cuanto mayor sea el valor de C, menor nmero de errores se permitirn, por lo que si $C \to \infty$ se considerara que todos los ejemplos son perfectamente separables. Cuanto ms pequeo sea C, mayor nmero de errores se admitiran de ejemplos mal clasificados.
  • epsilon ($\epsilon$): solo se considera en las SVM de regresin. Especifica los valores para los que no hay penalizacin asociada en la funcin de prdida del entrenamiento. El valor de $\epsilon$ afecta al nmero de vectores de soporte usados para construir la funcin de regresin. Cuando mayor sea el valor de $\epsilon$, menos vectores de soporte sern elegidos y ms ancho ser el canal.
  • nu ($\nu$): parmetro que controla el nmero de vectores de soporte y errores de entrenamiento.

Los parmetros que se espefican de manera fija son:

  • tol: Tolerancia mxima aceptada para converger.
  • max_iter: nmero mximo de iteraciones que se realizaran en caso de no haber convergido con la tolerancia especificada.

La formulacin matemtica para las SVM de regresin es la siguiente. Dados los vectores de entrenamiento $x_i \in \mathbb{R}^p, i = 1, ..., n$ y un vector $y \in \mathbb{R}^n$, $\epsilon-SVR$ resuelve el siguiente problema primal:

$$ min_{\omega, b, \xi^+, \xi^-} \frac{1}{2} \omega^T \omega + C \sum_{i=1}^n (\xi_{i}^+ + \xi_{i}^-)$$

sujeto a

$$ y_i - \omega^T \phi(x_i) - b \le \epsilon + \xi_{i}^+ $$ $$ \omega^T \phi(x_i) + b - y_i \lt \epsilon + \xi_{i}^- $$ $$ \xi_{i}^+ , \xi_{i}^- \ge 0, i = 1, ..., n $$

10.2. $\epsilon$-SVM con Kernel RBF ¶

Cuando se entrena una SVM con el kernel RBF, se han de tener en cuenta los parmetros C y gamma, entre los que existe un trade-off. El parmetro gamma define cunta influencia tiene un nico ejemplo del entrenamiento. Cuando ms grande sea gamma, ms cercanos han de estar otros ejemplos para ser afectados.

Si gamma es muy grande, el radio del rea de influencia de los vectores de soporte solo incluye al propio soporte y C no tendr ningn tipo de influencia. Por el contrario, cuando gamma es muy pequeo, el modelo queda muy limitado y no es capaz de ver toda la complejidad de los datos en el espacio de dimensiones. Cualquier vector de soporte incluir todo el dataset de entrenamiento.

En la siguiente grfica se puede observar dicho trade-off:

Por este motivo, para el GridSearchCV se han escogido unos valores de gamma de {0.001, 0.01, 0.1, 1, 10}, dado que la C podr tener los valores {0.001, 0.01, 0.1, 1, 5, 10, 100}.

La funcin del kernel radial es $exp(-\gamma \| x - x'\|^2)$, donde $\gamma$ es el parmetro gamma.
